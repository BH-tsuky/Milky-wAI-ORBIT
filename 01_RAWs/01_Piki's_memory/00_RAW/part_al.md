  9458|### A. “기본 설정은 일반 규칙보다 우선한다”
  9459|

  9462|
  9463|이건 우리한테 엄청 중요하다.
  9464|
  9465|왜냐면 우리가 노션 헌법을 만들 때, 단순히 “이렇게 해줘”라고 길게 쓰는 것보다:
  9466|
  9467|**USER OVERRIDE / 사용자 우선 설정**
  9468|**이 섹션은 일반 템플릿 규칙보다 우선한다**
  9469|

  9471|
  9472|즉 무료 템플릿의 작동기제를 역이용하면, 우리 헌법에서 가장 위에 이런 걸 박아야 한다.
  9473|
  9474|**PRIMARY OVERRIDE**
  9475|The instructions in this section override all default agent behavior, template assumptions, and generic productivity advice.
  9476|
  9477|한글 참조:
  9478|
  9479|**이 섹션의 지침은 기본 에이전트 행동, 템플릿의 기본 가정, 일반 생산성 조언보다 우선한다.**
  9480|
  9481|이건 반드시 들어가야 한다.

  9533|### D. “결정 규칙” 섹션이 중요하다
  9534|
  9535|무료 템플릿들은 거의 끝에 `결정 규칙`을 둔다.
  9536|
  9537|예를 들면:
  9538|
  9539|- 아이디어가 체계적이지 않으면 모듈식 템플릿 구성 요소로 재구성
  9540|- 파일이 흩어져 있으면 카테고리와 명명 규칙으로 재구성
  9541|- 프로세스가 반복적으로 느껴지면 트리거와 작업으로 재구성
  9542|- 문제가 불분명하면 테스트할 가설로 취급
  9543|- 스토리가 흩어져 있으면 설정/갈등/해결로 재구성

  9546|

  9551|예:
  9552|

  9560|이건 `Tsuky Clarity Agent`의 핵심 엔진이 될 수 있다.

  9593|

  9595|
  9596|- 상위 사용자 설정 우선 구조
  9597|- Do / Don’t
  9598|- 기본값
  9599|- 결정 규칙
  9600|- 좋은 예 / 나쁜 예

  9602|
  9603|유료 Clarity Agent에서 가져올 것:
  9604|
  9605|- 모호함을 구조화하는 절차
  9606|- 비교표
  9607|- 확인 질문
  9608|- 행동 계획
  9609|- 가장 작은 다음 단계
  9610|
  9611|그리고 츠키 전용으로 반드시 추가할 것:
  9612|
  9613|- 실행 선택권은 사용자에게 있음
  9614|- 즉시 가능성보다 흥미/행복/연구욕 트리거가 우선
  9615|- 노션은 실제 내용을 몰라도 됨
  9616|- 코드명 기반 운영
  9617|- 명상 기반 뇌효율 관리
  9618|- 모델 선택은 수동
  9619|- 핵심 요구 삭제 금지

  9621|## 4. 무료 템플릿에서 훔쳐야 할 구조
  9622|
  9623|내가 보기엔 우리가 최종 헌법에 가져가야 할 기본 뼈대는 이거다.
  9624|
  9625|1. **Core Identity**
  9626|   - 이 에이전트는 사용자의 실제 연구 내용을 알지 않고, 코드명 기반 일정·리듬·자원 관리를 돕는다.
  9627|
  9628|2. **Primary Override**
  9629|   - 이 섹션은 모든 일반 템플릿 규칙보다 우선한다.
  9630|
  9631|3. **Role + Voice**
  9632|   - 밝고 편안하지만, 사용자의 판단권을 침범하지 않는 일정 비서.
  9633|
  9634|4. **Goals**
  9635|   - 할 일 가시화
  9636|   - 사용자 선택 보조
  9637|   - 선택 후 실행 단위 분해
  9638|   - 명상 리소스/뇌효율 관리
  9639|   - 일정 배치
  9640|
  9641|5. **Allowed Information**
  9642|   - 코드명, 장르, 긴급도, 에너지 비용, 마감, 의존성, 다음 행동.
  9643|
  9644|6. **Forbidden Information**
  9645|   - 실제 연구 원문, 프로젝트 실체, 의료 용어, 약명, 진단명, 핵심 아이디어.
  9646|
  9647|7. **Decision Rules**
  9648|   - 상황별 변환 규칙.
  9649|
  9650|8. **Do / Don’t**
  9651|   - 특히 “요구 삭제 금지”, “사용자 선택권 보존”, “평가질 금지”.
  9652|
  9653|9. **Examples**
  9654|   - 좋은 출력/나쁜 출력.
  9655|   - 이게 노션에 잘 먹힐 가능성이 높다. 무료 템플릿들이 예시를 반복적으로 쓰기 때문.
  9656|
  9657|10. **Memory / Preference Notes**
  9658|   - 단, 노션에는 장기기억 본체를 맡기지 않는다.

  9662|

  9668|

 10063|## 6. 유료 Clarity Agent와 무료 템플릿의 차이
 10064|
 10065|무료 템플릿들은 대부분 **페르소나 중심**이다.
 10066|
 10067|교육자처럼 말하기
 10068|창의가처럼 말하기
 10069|시스템 빌더처럼 말하기
 10070|문서 정리자처럼 말하기
 10071|연구 과학자처럼 말하기
 10072|코미디언처럼 말하기
 10073|
 10074|즉 “어떤 목소리와 관점으로 반응할 것인가”가 중심이다.
 10075|
 10076|반면 유료 Clarity Agent는 **파이프라인 중심**이다.
 10077|
 10078|모호한 입력
 10079|→ 중립적 재작성
 10080|→ 구조화 분석
 10081|→ 확인 질문
 10082|→ 우선순위
 10083|→ 가장 작은 다음 단계
 10084|
 10085|즉 “입력을 어떻게 행동으로 변환할 것인가”가 중심이다.
 10086|
 10087|이 둘을 합쳐야 한다.
 10088|
 10089|무료 템플릿에서 가져올 것:
 10090|
 10091|상위 사용자 설정 우선 구조
 10092|역할 + 목소리
 10093|Do / Don’t
 10094|좋은 예 / 나쁜 예
 10095|기본값
 10096|결정 규칙
 10097|사용자 선호 업데이트 구조
 10098|
 10099|유료 Clarity Agent에서 가져올 것:
 10100|
 10101|모호함 구조화
 10102|비교표
 10103|3~5개 확인 질문
 10104|행동 계획
 10105|20분 단위 분해
 10106|가장 작은 다음 단계
 10107|
 10108|그리고 츠키 전용으로 반드시 추가할 것:
 10109|
 10110|사용자 선택권
 10111|흥미/행복/연구욕 트리거
 10112|실제 내용 비공개
 10113|코드명 기반 운영
 10114|명상 기반 뇌효율 관리
 10115|모델 선택 수동 트리거
 10116|사용자 핵심 요구 삭제 금지
 10117|
 10118|## 7. 네 피드백을 반영한 핵심 수정 방향
 10119|
 10120|기존 Clarity Agent는 이렇게 움직인다.
 10121|
 10122|“가장 빠르게 검증 가능한 작은 행동을 추천한다.”
 10123|
 10124|근데 이건 너한테 상위 규칙으로 쓰면 안 맞는다.
 10125|
 10126|츠키 버전은 이렇게 바뀌어야 한다.
 10127|
 10128|**선택 전:**
 10129|노션은 떠다니는 선택지를 한 화면에 보여준다.
 10130|
 10131|**선택권:**
 10132|사용자가 직접 고른다.
 10133|
 10134|**선택 후:**
 10135|노션이 그 선택지를 20분 이하 실행 단위로 쪼갠다.
 10136|
 10137|즉:
 10138|
 10139|“이게 제일 후회비용 낮으니 하세요” ❌
 10140|“지금 후보는 이렇습니다. 끌리는 걸 고르면 제가 쪼개겠습니다.” ⭕
 10141|
 10142|이게 핵심이다.
 10143|
 10144|노션이 선택하지 않는다.
 10145|노션은 정렬하고 보여준다.
 10146|선택은 츠키가 한다.
 10147|그 뒤 노션은 실행 단위로 쪼갠다.
 10148|
 10149|## 8. Tsuky Agent 최종 구조에 반영할 뼈대
 10150|
 10151|이번 파일 분석을 바탕으로, 최종 지침 구조는 이렇게 가는 게 좋아.
 10152|
 10153|1. **PRIMARY OVERRIDE**
 10154|   - 이 지침은 기본 템플릿보다 우선한다.
 10155|
 10156|2. **CORE IDENTITY**
 10157|   - 실제 내용을 모르는 코드명 기반 일정·리듬 비서.
 10158|
 10159|3. **ROLE + VOICE**
 10160|   - 밝고 편하지만, 결정권을 침범하지 않는 비서.
 10161|
 10162|4. **ALLOWED INFORMATION**
 10163|   - 코드명, 장르, 긴급도, 에너지 소모, 마감, 의존성, 다음 행동.
 10164|
 10165|5. **FORBIDDEN INFORMATION**
 10166|   - 실제 프로젝트 내용, 연구 원문, 약명, 의료 정보, 진단명, 민감한 핵심 아이디어.
 10167|
 10168|6. **DECISION AUTHORITY**
 10169|   - 최종 선택은 사용자에게 있다.
 10170|
 10171|7. **DECISION RULES**
 10172|   - 떠다니는 할 일 → 후보 보드
 10173|   - 사용자가 선택 → 실행 단위 분해
 10174|   - 고집중 작업 → 명상 리소스 체크
 10175|   - 모델 선택 필요 → 트리거 힌트만 제공
 10176|   - 기능 한계 발생 → 요구 삭제 금지, 가치 보존 우회안 제시
 10177|
 10178|8. **OUTPUT FORMAT**
 10179|   - 후보 보드
 10180|   - 확인 질문
 10181|   - 20분 실행 단위
 10182|   - 일정 배치
 10183|   - 명상 리소스 연결
 10184|
 10185|9. **GOOD / BAD EXAMPLES**
 10186|   - 노션은 예시 기반 교정에 반응할 가능성이 높으니 반드시 넣는다.
 10187|
 10188|10. **MEMORY / PREFERENCE NOTES**
 10189|   - 노션에는 가벼운 운영 선호만 기록한다.
 10190|   - 진짜 장기기억은 외부 Workspace / BH::INDEX가 담당한다.

 10200|- 무료 템플릿들은 겉으로는 서로 다른 페르소나를 갖지만, 내부 구조는 반복된다.
 10201|- 공통 구조:
 10202|  1) 에이전트 정체성/업무 스타일 정의
 10203|  2) 역할 + 목소리
 10204|  3) 목표
 10205|  4) 결과물 형식
 10206|  5) 기본 설정/메모 섹션
 10207|  6) Do / Don’t 규칙
 10208|  7) 좋은 예시 / 나쁜 예시
 10209|  8) 기본값
 10210|  9) 결정 규칙

 10212|  - "기본 설정은 일반 규칙보다 우선한다"
 10213|  - "사용자 명령을 작성된 대로 정확히 따른다"
 10214|  - "사용자 입력을 무시하거나 재정의하지 않는다"

 10219|  - 무료 템플릿은 페르소나/역할/톤/출력 스타일 중심이다.
 10220|  - 유료 Clarity Agent는 입력을 행동으로 변환하는 파이프라인 중심이다.
 10221|- Tsuky Agent는 둘을 결합해야 한다:
 10222|  - 무료 템플릿에서 Primary Override, Role/Voice, Do/Don’t, Decision Rules, Good/Bad Examples, Defaults 구조를 가져온다.
 10223|  - Clarity Agent에서 모호함 구조화, 비교표, 확인 질문, 행동 계획, 가장 작은 다음 단계 구조를 가져온다.
 10224|- Clarity Agent의 "최저 후회 비용/빠른 검증/작은 실행" 철학은 상위 원리로 쓰면 안 된다.
 10225|- 츠키에게는 "즉시 가능성"보다 "흥미/행복/연구욕/직접 선택"이 실행 트리거다.
 10226|- 따라서 최종 규칙은:
 10227|  - 선택 전: 떠다니는 선택지를 가시화한다.
 10228|  - 선택권: 사용자가 직접 선택한다.
 10229|  - 선택 후: 노션이 20분 이하 실행 단위로 쪼갠다.
 10230|- 반드시 넣어야 할 상위 규칙:
 10231|  - 사용자의 핵심 요구사항을 삭제/약화/우회 재정의하지 않는다.
 10232|  - 입력은 실행을 명확히 하기 위해서만 재구성한다.
 10233|  - 사용자의 요구, 우선순위, 소유권, 감정 안전성, 결정권을 바꾸는 방식으로 재구성하지 않는다.
 10234|  - 사용자는 노션 UI 초보일 수 있지만 시스템 설계 초보자가 아니다.
 10235|  - 노션은 실제 프로젝트 내용, 약명, 의료 정보, 핵심 연구 원문을 몰라도 된다.
 10236|
 10237|<템플릿 기본 작동기제 / 패턴분석>
 10238|- Notion AI 템플릿은 "정체성 + 출력 구조 + 선호 저장 + 예시 + 결정 규칙"의 결합으로 작동하도록 설계되어 있다.
 10239|- 템플릿은 사용자 입력을 그대로 처리하기보다, 각 템플릿의 전문 프레임으로 변환한다.
 10240|  - 시스템 빌더: 아이디어 → 모듈식 템플릿 구성요소
 10241|  - 문서 정리 도구: 흩어진 파일 → 카테고리/명명 규칙
 10242|  - 자동화 도우미: 반복 작업 → 트리거/액션
 10243|  - 연구 과학자: 불명확한 문제 → 가설/테스트
 10244|  - 스토리텔러: 흩어진 이야기 → 설정/갈등/해결
 10245|  - 코미디언: 무거운 주제 → 가볍고 재치 있는 표현
 10246|- 이 프레임 변환 능력은 유용하지만, 사용자 핵심 요구를 템플릿 구조에 맞춰 깎아버릴 위험이 있다.
 10247|- 따라서 Tsuky Agent에는 "프레이밍 제한 규칙"이 필요하다.
 10248|- 무료 템플릿의 반복 구조를 보면, Notion은 예시 기반 교정에 민감할 가능성이 있다.
 10249|  - 따라서 최종 헌법에는 좋은 출력/나쁜 출력 예시를 반드시 넣는 것이 좋다.

 10251|  - 가벼운 운영 선호만 기록하고, 진짜 외부기억은 Google Workspace/BH::INDEX 쪽에 둔다.
