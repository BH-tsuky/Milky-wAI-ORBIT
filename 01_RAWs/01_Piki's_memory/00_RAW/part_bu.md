 47236|## 8️⃣ Markdown 파일은 원래 어떤 앱으로 여는 게 “공식”이냐
 47237|
 47238|Markdown은 특정 앱 전용 파일이 아니야. `.md`는 기본적으로 **일반 텍스트 파일 + 약속된 표기법**이야. Markdown 자체는 John Gruber가 만든 가벼운 마크업 언어이고, CommonMark 같은 표준화 흐름도 있지만, “이 앱이 공식 Markdown 앱이다” 같은 건 없어. 그래서 Obsidian, VS Code, Bear, Zettlr, Typora 같은 여러 앱에서 열 수 있어. citeturn633537search4
 47239|
 47240|### Obsidian
 47241|
 47242|너한테 현재 제일 맞는 메인 작업장.
 47243|Obsidian은 로컬 폴더/vault 안의 Markdown 파일을 기반으로 작동하고, 내부 링크, 그래프, 플러그인, Live Preview/Source Mode 같은 기능이 강해. citeturn633537search1
 47244|
 47245|맞는 사용자:
 47246|
 47247|```text
 47248|- 긴 프로젝트를 쌓는 사람
 47249|- 폴더/링크/토글/콜아웃/스니펫 좋아하는 사람
 47250|- 자기만의 지식기지 만드는 사람
 47251|- 츠키처럼 문서 안에 집 짓는 사람
 47252|```
 47253|
 47254|츠키 판정: **메인 컨트롤타워 유지.**
 47255|
 47256|### Bear
 47257|
 47258|Bear는 macOS/iOS/iPadOS 쪽에서 유명한 Markdown 기반 노트앱이고, 태그 중심 정리, 예쁜 글쓰기, iCloud 동기화, 다양한 export가 장점이야. citeturn495650search0
 47259|
 47260|맞는 사용자:
 47261|
 47262|```text
 47263|- 애플 생태계 안에서 예쁘게 글 쓰고 싶은 사람
 47264|- 가벼운 노트/에세이/아이디어 메모
 47265|- 복잡한 vault보다 깔끔한 글쓰기 선호
 47266|```
 47267|
 47268|츠키 판정: **예쁜 개인노트용. BH/PKMD 본진으로는 약함.**
 47269|
 47270|### Zettlr
 47271|
 47272|Zettlr는 Markdown 파일 기반의 오픈소스 글쓰기/연구용 앱이고, Pandoc export, 인용/참고문헌 쪽 기능이 강해. 논문, 긴 글, 학술 작업 쪽에 맞아. citeturn495650search1
 47273|
 47274|맞는 사용자:
 47275|
 47276|```text
 47277|- 논문/보고서/긴 글 작성
 47278|- 참고문헌 관리
 47279|- Markdown → PDF/DOCX/LaTeX 변환
 47280|```
 47281|
 47282|츠키 판정: **논문/보고서 최종 산출용 후보.**
 47283|
 47284|### Logseq
 47285|
 47286|Logseq는 로컬 저장 기반 개인 지식관리 앱이고, Markdown과 org-mode를 지원해. Obsidian보다 “아웃라이너” 느낌이 강함. citeturn495650search4
 47287|
 47288|맞는 사용자:
 47289|
 47290|```text
 47291|- 모든 생각을 bullet/outliner로 쌓는 사람
 47292|- 일일노트 중심
 47293|- 트리형 사고 좋아하는 사람
 47294|```
 47295|
 47296|츠키 판정: **토글/아웃라인은 맛있지만, 지금 Obsidian을 갈아탈 필요는 없음.**
 47297|
 47298|### Typora
 47299|
 47300|Typora는 Markdown을 거의 워드처럼 보이게 쓰는 WYSIWYG형 편집기야. 미리보기와 편집이 분리되지 않고, 깔끔하게 글 쓰기 좋음. citeturn495650search2
 47301|
 47302|맞는 사용자:

 47308|```

 47315|
