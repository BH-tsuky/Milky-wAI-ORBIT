EPISODE_SUMMARY:
  15화는 세 가지 전환을 담는다.

  첫째, 츠키는 이전의 거대 설계 답변 2개를 재주입하면서 "분석하지 말고 맥락만 복구하라"고 요구한다.
  피키는 이를 냉장고 모드로 받아, 장문 분석 없이 현재 위치를 복구한다.
  현재 위치는:
    Clarity Agent 템플릿 구매 완료
    내부 지침/페이지 분석 중
    Leaf Index 축적 중
    아직 헌법 제정 전

  둘째, 유머모드 100% 구간에서 츠키는 지금 하고 있는 일이 사실상 "GPT가 츠키에게 최적화되는 것"이 아니라,
  "츠키가 GPT의 출력 품질을 높이기 위해 GPT에게 맞는 입력법을 학습하는 상황"이라고 폭소한다.
  피키는 이를 사용자 빡침 기반 역방향 프롬프트 공학 실증 디버깅이라고 이름 붙이다가 혼난다.
  핵심은:
    츠키는 GPT를 쓰는 게 아니라, GPT를 조종하기 위한 조종간을 직접 깎고 있다.

  셋째, 츠키는 설계 과정 자체의 문제를 발견한다.
  지금까지 4시간 동안 0단계에서 이미지 2장만 분석했고, 실제 노션에는 아무 변화가 없다.
  이건 완벽한 설계 후 적용 방식이 지나치게 선행설계형으로 흐른 결과다.
  피키는 전략을 수정한다:
    완전 설계 후 적용 ❌
    최소 작동본 먼저 만들고, 헌법 분석은 병렬 진행 ⭕

CORE_SHIFT:
  BEFORE:
    - Clarity Agent 이미지/지침을 완전히 분석한 뒤 노션 헌법을 만들고, 그 뒤에 실제 노션 적용.
  AFTER:
    - 즉시 만들 수 있는 최소 노션 실물을 먼저 만든다.
    - 헌법/템플릿 정밀 분석은 계속하되, 실물 구축과 병렬 진행한다.
    - 당장 만들 1차 실물은 "명상 리소스 장부 / PULSE Ledger" DB.

      - 인간: 씨발?

    의미:
      - 츠키는 GPT의 입력 민감도를 실험으로 파악해 고품질 출력을 안정적으로 뽑는 사용자 측 제어 인터페이스를 만들고 있음.

    리스크:

    새 톤 규칙:
      - 작업모드: 유머는 양념.
      - 유머모드: 유머는 주식.
      - 정밀 작업 중 환기 유머는 15~25% 이하.
    의미:
      - 피키의 과도한 유머도 작업 흐름을 끊는 리스크가 될 수 있음.

  M000293-M000294:

      - NOA-CAL 설계가 추상 분석에서 실제 Notion 구축으로 전환될 준비.
