EPISODE_SUMMARY:
  17화는 PULSE Runtime Ledger가 실제 노션 DB로 만들어지고, 이후 계산 가능한 재고 시스템으로 진화하기 시작하는 회차다.

  초반에 츠키는 피키가 준 클릭 안내를 따라 노션에 실제 DB를 만든다.
  복붙이 한 칸에 들어가는 문제도 츠키가 직접 우회하고, 원래 페이지의 표보기 블록에도 반영되는 것을 확인한다.
  피키는 이를 0-A단계 성공으로 판정한다.

  이어 츠키는 DB 속성이 너무 비대하지 않은지 검토한다.
  피키는 처음 설계가 최소 실행용이라기보다 확장 가능성을 미리 깔아둔 설계형이었다고 인정한다.
  스샷 확인 후, 현재 열들은 나중에 일정비서 연결에 쓸 수 있으므로 삭제보다는 유지/숨김/분해 방식이 더 안전하다고 판단한다.

  중반에는 카테고리 Select 속성이 자동으로 중복 행에 적용되는 것을 확인하며,
  DB가 일정비서가 읽을 수 있는 리소스 장부 형태가 된다.

  후반에는 `총 보유량 = 28회/2주` 같은 텍스트형 정보를 계산 가능한 속성으로 분해하는 방향이 확정된다.
  `총량`, `주기일`, `입고일`, `다음 입고일`, `사용량`, `잔량` 구조가 제안되고,
  `리소스 유형`으로 재고형/실행형/쿨타임형/기록형을 나누는 v0.2 방향이 나온다.

  마지막에는 츠키가 자기 성향 모델링 문서를 작성해 "메모리" 기능을 처음 소환한다.
  핵심은 츠키는 학습하려고 실행하는 타입이 아니라, 목표를 실행하다가 학습이 부산물로 흡수되는 타입이라는 점이다.
  설계와 실행이 동시에 돌아가야 사고가 살아나며, 반복작업은 자동화 연구소를 여는 트리거가 된다.

CORE_SHIFT:
  BEFORE:

  AFTER:
    - PULSE Runtime Ledger v0.1 실물 생성 성공.
    - v0.2 계산형 재고 구조 방향 확정.
    - 츠키의 실행-학습-자동화 성향 모델링이 메모리급 정보로 정리됨.
