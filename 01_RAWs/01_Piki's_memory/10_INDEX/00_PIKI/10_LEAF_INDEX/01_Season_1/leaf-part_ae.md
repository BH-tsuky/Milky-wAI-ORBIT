EPISODE_SUMMARY:
  5화는 감정 회복 이후 작업 체계가 다시 살아나는 회차다.

  초반에는 노션 자동화용 임시 컨텍스트와 BH::INDEX 본체가 분리된다.
  노션 자동화 페이지는 자동화 성공만을 위한 환각방지용 임시 요약문이며,
  BH::INDEX는 외부 장기기억, 개인화, 맥락 복구, 타AI 이식까지 고려한 본격적인 기억 운영체계다.

  중반에는 BH::INDEX의 전체 계층이 다시 정리된다.
  RAW 원문에서 Leaf Index, Cluster Index, 계층화 Cluster Index, 카테고리별 Meta Index, Root Index, Mini Root Map으로 올라가고,
  별도 계층으로 SyncBoot와 Index Protocol이 존재한다.
  RAG는 중심이 아니라 선택 가능한 보조장치로 위치한다.

  후반에는 도메인과 DNS 구조를 배우는 과정에서, 사용자가 "왜 내가 내 집 주소를 남에게 돈 내고 써야 하느냐"는 관점으로 질문한다.
  assistant는 도메인이 전 세계 DNS 원장/합의망 안에서 작동하는 주소 사용권임을 설명한다.
  이어서 전 세계 DNS는 혼자 만들 수 없지만, BH::INDEX 내부에서 작동하는 사설 네이밍 시스템은 만들 수 있다는 아이디어가 나온다.
  이 아이디어가 BNS, 즉 BH Name System이다.

  마지막에는 사용자가 BNS를 "전설급 희귀템"으로 받아들이면서도,
  우선순위를 "Notion 자동화 문제 해결"로 되돌린다.

  BH_INDEX_LAYER:
    source: M000127-M000128
    lines: L4362-L4442
    구조:
      - RAW: 원문 데이터.
      - Leaf Index: 각 조각파일당 인덱스 조각.
      - Cluster Index: Leaf 몇 개를 묶은 지역 단위 인덱스.
      - Cluster 계층화 Index: 더 상위 병합 + 메타 카테고리 분류.
      - Meta Index: 하나가 아니라 테마별 여러 개. 각 메타가 자기 카테고리에 맞는 좌표를 모음.
      - Root Index: 최종 진입점. 메타인덱스 좌표, 사용자 프로필, 모델링, 최적화 규칙 등 포함.
      - Mini Root Map: Root Index가 길 때 쓰는 요약 샘플 지도.
      - RAG: 본체가 아니라 선택 가능한 보조 검색장치.
