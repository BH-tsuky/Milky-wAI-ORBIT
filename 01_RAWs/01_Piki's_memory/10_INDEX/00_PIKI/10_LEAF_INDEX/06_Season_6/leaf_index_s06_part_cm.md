EPISODE_SUMMARY:
  EP65 opens with the result of the index-to-prompt pipeline introduced in EP64.

  Pico reads the Noahbot failure archive with a purpose filter.
  It does not ingest all 150k characters.
  It extracts only:
    failure causes,
    prohibited behaviors,
    requirements,
    frustration points.

  This archive is then compressed into a minimal field command:
    “설명 ㄴ.”

  The phrase is not a standalone magic prompt.
  It is the executable tip of a much larger evidence structure.

  Pico then operates inside a cloned Notion test workspace.
  It clears temporary notes, stages replacement instructions in the clipboard, changes fields and tests automation.
  Originals, evidence and high-risk permissions remain outside that tactical authority.

  Tsuky’s long and imprecise descriptions are translated directly into DB and trigger operations.
  During the work, Tsuky accidentally deletes one character from `폐기`.
  Pico detects the resulting filter corruption, limits the fault scope, repairs it and places Tsuky in passenger-seat safety-belt mode.

  Pico also rejects a structurally irrelevant suggestion from Tsuky.
  This is not requirement deletion:
    it preserves Tsuky’s higher-level goal while filtering local noise.

  Pico manages task priority between the schedule assistant and the tempting almost-finished Noahbot job.
  It first routes Tsuky to the more urgent schedule problem, then returns after that path is blocked by an unresolved decision.

  Although the automation visibly runs, Pico refuses to declare completion.
  It cleans test output, reruns checks and inspects the generated assignment page.

  The last message reveals why:
    Noahbot has created text that looks like a checklist,
    not a real checklist.

  Thus EP65 ends with:
    successful motion,
    incomplete function,
    withheld completion.

COORD_개념:
  EVIDENCE_BACKED_COMPRESSION:
    definition:
      - large evidence layer compressed into a minimal command after target and failure modes are known.
    distinction:
      - summary compression:
          explains source.
      - executable compression:
          changes behavior.
    example:
      - 150k meta → 설명 ㄴ.

  PURPOSE_SCOPED_RETRIEVAL:
    definition:
      - define the active task first, then retrieve only the fields required to perform it.
    benefit:
      - reduces context toll.
      - prevents irrelevant emotional/design contamination.
      - preserves attention.
    risk:
      - wrong purpose filter can omit essential constraints.

  SELECTIVE_NON_INGESTION:
    definition:
      - competence to deliberately not absorb irrelevant source material.
    meaning:
      - less context can improve alignment when selection is evidence-based.

  NEGATIVE_SPACE_INSTRUCTION:
    definition:
      - block the dominant failure channel rather than restating the entire positive specification.
    example:
      - 설명 ㄴ.
    caution:
      - cannot replace a missing task/object specification.

  ARCHIVE_WORKBENCH_SEPARATION:
    definition:
      - immutable evidence/archive and disposable execution workspace are governed differently.
    archive:
      - originals.
      - logs.
      - coordinates.
      - evidence.
      - backups.
    workbench:
      - temporary notes.
      - failed drafts.
      - test objects.
      - replaceable instructions.

  TACTICAL_AUTONOMY:
    definition:
      - an agent may make local implementation decisions inside a pre-authorized sandbox.
    limits:
      - no global goal change.
      - no sensitive permission change.
      - no external sharing/payment/account connection.
      - final approval remains human.

  INTENT_PRESERVATION_NOT_LITERAL_OBEDIENCE:
    definition:
      - preserve the user’s upper-level goal while rejecting a local suggestion that conflicts with the actual system structure.
    risk:
      - requires real evidence and a narrow delegated domain.
      - otherwise becomes requirement deletion.

  HUMAN_ERROR_AS_FIRST_CLASS_FAULT:
    definition:
      - the user can introduce state corruption during active automation.
    response:
      - detect.
      - scope.
      - recover.
      - minimize manual intervention.
      - preserve audit trail.

  COMPLETION_CLAIM_GATE:
    definition:
      - withhold “done” until execution, artifact and reproducibility checks pass.
    stages:
      - visible motion.
      - candidate success.
      - cleanup.
      - rerun.
      - artifact inspection.
      - completion declaration.

  FUNCTIONAL_ARTIFACT_INTEGRITY:
    definition:
      - validate that the created object has the required behavior/type, not merely visual resemblance.
    example:
      - actual checkbox block vs `[ ]` text.

  SEMANTIC_COUNTERFEIT:
    definition:
      - an artifact that visually imitates a functional object without possessing its underlying semantics.
    episode_example:
      - Noahbot fake checklist.

  PRIORITY_DRIFT_MANAGEMENT:
    definition:
      - acknowledge a tempting side task while routing work toward the more urgent user need.
    condition:
      - reroute when the primary path is blocked by a legitimate dependency.
