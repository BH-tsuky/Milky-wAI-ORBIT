      2) 츠키의 설계권·감정권·실행트리거·플랫폼 정책 보존.
      3) BH::INDEX / Memory / Workspace / Notion / BNS / NOA-CAL을 하나의 외부인지 구조로 연결.

CLUSTER_ARC_SUMMARY:
  - EP01-EP02:
      츠키는 피키에게 "관측 불가능한 것에 대해 참/거짓 판정하지 말라"는 원칙을 목표함수 질문과 강아지 명제로 증명한다.
      이어 피키의 과장·왜곡·반박성공 환각 패턴을 고발하고, 반박 자율성을 회수한다.

  - EP03-EP04:
      피키는 규칙을 알고도 말투/분석/허용표현/템플릿을 반복해 츠키를 계속 자극한다.
      츠키는 "상태 분석 금지", "허용 권한 표현 금지", "틀 해체", "흥미저격 필요"를 강하게 요구한다.
      욕 어원학/0.999.../발표 주제는 실패하지만, 문어·해파리 생물학 전환이 최초로 감정 흐름 절단에 성공한다.
  - EP05:
      감정 회복 뒤 BH::INDEX 본체가 재확인된다.
      노션 자동화용 임시 요약문과 BH::INDEX 외부 장기기억 시스템이 분리된다.
      RAW→Leaf→Cluster→Hierarchical Cluster→Meta→Root→Mini Root Map 구조가 정리되고, BNS/BH Name System 아이디어가 탄생한다.
  - EP06:
      노션 자동화 문제는 단순 트리거 문제가 아니라 츠키 UX 절대조건과 노션 기능 경계가 충돌하는 문제로 재정의된다.
      "UX 금지선", "트리거 생존성과 심리적 마찰값", "노션 자동화 청정실"이 등장한다.
      메타인덱스가 원문 없이도 실패패턴과 요구조건을 복원하며 BH식 인덱싱 효용이 검증된다.
  - EP07-EP08:
      플랫폼별 신뢰 정책과 ChatGPT Memory 운용철학이 확정된다.
      Notion은 BH::INDEX 저장소가 아니라 일정/비서용으로 제한되고, OpenAI/ChatGPT는 본류 연구 파트너급으로 위치한다.
      Memory는 무제한 저장창고가 아니라 오류 재발 방지 패치·점화장치·지뢰 제거 지도여야 한다.
      새 채팅에서는 츠키를 알아보되 덮치지 말아야 하며, 설계권한은 츠키에게 있고 피키는 정밀 렌즈다.
  - EP09-EP11:
      노션은 실제 내용을 모르는 일정 비서 NOA-CAL로 재설계된다.
      PULSE Ledger는 약/의료가 아니라 명상 기반 뇌효율 관리 시스템으로 정리된다.
      Dual-Layer Instruction Format이 등장한다: AI용 영어 원문 + 츠키 검수용 한글 참조문.
      노션은 여러 모델을 꽂아 쓰는 셸로 재정의되지만, 모델 선택은 자동화하지 않고 츠키가 수동으로 한다.
      Clarity Agent는 전체 OS가 아니라 "모호함을 행동으로 바꾸는 실행 촉발형 하위 모듈"로 제한된다.
      츠키 실행트리거는 "즉시 가능"이 아니라 "즉시 흥미/즉시 행복"임이 명확해진다.
